# Nifty 100 Financial Intelligence — ETL Pipeline & BI Dashboards

A full-stack financial intelligence platform for the **Nifty 100** stock universe.
It extracts data from Excel, runs an ETL pipeline into PostgreSQL, scores companies
with ML health metrics, and serves 7 interactive BI dashboards via a Django REST API.

## Project Overview

| Component | Technology |
|---|---|
| Data source | Excel files (Screener.in exports) |
| Data warehouse | PostgreSQL 15 (via Docker) |
| ETL pipeline | Python + Pandas + SQLAlchemy |
| ML scoring | scikit-learn + custom scoring engine |
| REST API | Django 4.2 + Django REST Framework |
| Task queue | Celery + Redis |
| Dashboards | 7 × HTML/JS (Chart.js) served statically |
| Stock price data | Yahoo Finance via `yfinance` |

**90 Nifty 100 companies** are scored across 6 dimensions:
Profitability · Growth · Leverage · Cashflow · Dividend · Trend

Health labels: `EXCELLENT` · `GOOD` · `AVERAGE` · `WEAK` · `POOR`

```

        │
        ▼

  ┌─────────────────────────────────────┐

  └─────────────────────────────────────┘
        │
        ▼

        │
        ▼

        │
        ▼
  7 × BI Dashboards (port 5500)
```

## Quick Start — First Time Setup

> Run every command from **PowerShell** or **Command Prompt**.
> Do NOT use a virtual environment — run directly from Anaconda Python.

### Terminal 1 — Backend

```powershell

# Step 2 — Start PostgreSQL + Redis via Docker

# Step 3 — Install all Python dependencies

python etl/01_extract_from_excel.py
python etl/02_clean_and_transform.py
python etl/03_load_to_warehouse.py
python etl/04_ml_scores.py
python etl/05_compute_analysis.py

# Step 6 — Start Django API server — KEEP THIS TERMINAL OPEN
python manage.py runserver 8000
```

Open a **new PowerShell window**:

```powershell

cd "C:\Users\dudde\OneDrive\Documents\Nifty-100-Financial-Intelligence-ETL-Pipeline\dashboards"

# Step 8 — Serve dashboards — KEEP THIS TERMINAL OPEN

```

```
http://localhost:5500/01_executive_overview.html
```

## ETL Pipeline Details

### Step 1 — Extract (`01_extract_from_excel.py`)
Reads all Excel files from `data/source/` and writes raw CSV files to `data/raw/`.
Handles multi-sheet Screener.in export format.

### Step 2 — Transform (`02_clean_and_transform.py`)
- Normalises column names and data types
- Handles missing values and outliers
- Writes clean CSVs to `data/clean/`

### Step 3 — Load (`03_load_to_warehouse.py`)
Loads clean CSVs into PostgreSQL dimension and fact tables:
`dim_company`, `dim_year`, `fact_profit_loss`, `fact_balance_sheet`, `fact_cash_flow`

### Step 4 — ML Scores (`04_ml_scores.py`)
Computes 6-dimension health scores (0–100) for all companies:

| Dimension | Weight | Key Metrics |
|---|---|---|
| Profitability | 25% | NPM, OPM, Interest Coverage |
| Growth | 20% | Sales CAGR, Profit CAGR |
| Leverage | 15% | Debt/Equity, Equity Ratio |
| Cashflow | 20% | OCF, FCF, Cash Conversion |
| Dividend | 10% | Payout %, Consistency |
| Trend | 10% | 3Y Revenue & Profit slope |

Health label bands: EXCELLENT ≥ 80 · GOOD ≥ 60 · AVERAGE ≥ 40 · WEAK ≥ 20 · POOR < 20

Also generates `fact_pros_cons` with ML-driven strengths/weaknesses per company.

### Step 5 — CAGR Analysis (`05_compute_analysis.py`)
Computes per-period (10Y / 5Y / 3Y / TTM) metrics for all companies:
- **Sales CAGR%** — from P&L
- **Profit CAGR%** — from P&L
- **ROE%** — computed per period using cumulative retained earnings proxy
- **Stock Price CAGR%** — fetched from Yahoo Finance (NSE tickers via `yfinance`)

Writes to `fact_analysis`. Takes ~3–5 minutes on first run (downloads stock prices).

> Re-run `04_ml_scores.py` after running `05_compute_analysis.py` to include
> the updated CAGR data in health scores.

## Database Schema

All tables live in the `public` schema of the `nifty100` PostgreSQL database.

### Dimension Tables

| Table | Description |
|---|---|
| `dim_company` | Master list of 90+ companies (symbol, name, sector, ROE, ROCE) |
| `dim_year` | Fiscal years + TTM flag + sort order |
| `dim_sector` | Sector master |
| `dim_health_label` | EXCELLENT / GOOD / AVERAGE / WEAK / POOR |

### Fact Tables

| Table | Description | Key Fields |
|---|---|---|
| `fact_profit_loss` | Annual P&L per company | `sales`, `net_profit`, `opm_pct`, `eps`, `dividend_payout_pct` |
| `fact_balance_sheet` | Annual BS per company | `borrowings`, `reserves`, `total_assets`, `debt_to_equity` |
| `fact_cash_flow` | Annual CF per company | `operating_activity`, `investing_activity`, `free_cash_flow` |
| `fact_analysis` | CAGR by period | `compounded_sales_growth_pct`, `stock_price_cagr_pct`, `roe_pct` |
| `fact_ml_scores` | ML health scores | `overall_score`, `profitability_score`, `growth_score`, `health_label` |
| `fact_pros_cons` | ML-generated insights | `is_pro`, `text`, `confidence` |
| `fact_stock_price` | Monthly closing prices | `price_date`, `close_price` (created by step 5) |

## Troubleshooting

### Charts show no data / "Cannot reach API"
Django is not running. Make sure Terminal 1 shows `Starting development server at http://127.0.0.1:8000/`.
Test with: `curl http://localhost:8000/api/companies/`

### CORS error in browser console
You opened the HTML file directly (`file:///...`) instead of via the server.
Always use: `http://localhost:5500/01_executive_overview.html`

### `No module named 'django'`
You are in a virtual environment that doesn't have Django. Run `deactivate` first, then retry.

### Database connection refused
Docker is not running or Postgres hasn't started yet.
```powershell
docker-compose up -d postgres redis
docker-compose ps   # both should show "healthy"
```

### ETL script fails / no data in dashboards
Make sure your Excel files are in `data/source/` before running ETL.
Check: `python manage.py shell -c "from intelligence.models import DimCompany; print(DimCompany.objects.count())"`
Should print `90` (or similar).

### ml-score returns empty / Health score 0.0
Run `python etl/04_ml_scores.py` — it must complete with "ML scoring complete."
Check: `python manage.py shell -c "from intelligence.models import FactMlScore; print(FactMlScore.objects.count())"`

### Stock CAGR shows "not available"
Run `pip install yfinance` then `python etl/05_compute_analysis.py`.
Requires internet connection. Takes 3–5 minutes.

### `source` command not found on Windows
Use `.venv\Scripts\activate` instead of `source .venv/bin/activate`.
Or just skip the venv entirely and use Anaconda Python directly.
