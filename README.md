## Quick Start (Local)

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Start PostgreSQL + Redis via Docker
```bash
docker-compose up postgres redis -d
```

### 3. Run the ETL pipeline
```bash
# Extract raw data from Excel files
python etl/01_extract_from_excel.py

# Clean and transform
python etl/02_clean_and_transform.py

# Load into PostgreSQL warehouse
python etl/03_load_to_warehouse.py

# Run ML health scoring
python etl/04_ml_scores.py
```

### 4. Start Django
```bash
# Create Django project (first time only)
django-admin startproject nifty100 .
python manage.py startapp intelligence

# Copy models.py, serializers.py, views.py, urls.py, tasks.py into intelligence/
# Copy settings.py into nifty100/
# Copy celery.py into nifty100/

python manage.py migrate --run-syncdb
python manage.py runserver
```

### 5. API is live at
- Swagger UI:  http://localhost:8000/api/docs/
- Redoc:       http://localhost:8000/api/redoc/
- Companies:   http://localhost:8000/api/companies/
- Leaderboard: http://localhost:8000/api/leaderboard/

## Key API Endpoints

| Method | URL | Description |
|--------|-----|-------------|
| GET | `/api/companies/` | List all companies (paginated, filterable) |
| GET | `/api/companies/{symbol}/` | Company details |
| GET | `/api/companies/{symbol}/full/` | Full snapshot with all financials |
| GET | `/api/companies/{symbol}/profit-loss/` | P&L history |
| GET | `/api/companies/{symbol}/balance-sheet/` | Balance sheet history |
| GET | `/api/companies/{symbol}/cash-flow/` | Cash flow history |
| GET | `/api/companies/{symbol}/analysis/` | Growth analysis (3Y/5Y/10Y) |
| GET | `/api/companies/{symbol}/ml-score/` | ML health scores |
| GET | `/api/leaderboard/?n=20&sector=IT` | Top companies by score |
| GET | `/api/sector-summary/` | Sector-wise averages |
| GET | `/api/compare/?symbol=TCS&symbol=INFY` | Side-by-side comparison |

## Power BI Connection

1. Open Power BI Desktop
2. Get Data → PostgreSQL
3. Server: `localhost:5433`
4. Database: `nifty100`
5. Connect to these tables:
   - `dim_company`, `dim_year`, `dim_sector`
   - `fact_profit_loss`, `fact_balance_sheet`, `fact_cash_flow`
   - `fact_analysis`, `fact_ml_scores`

Use the star schema relationships:
- `fact_*` tables join to `dim_company` on `symbol`
- `fact_*` tables join to `dim_year` on `year_id`
