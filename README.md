# Nifty 100 Financial Intelligence — Setup Guide

## Overview

This repository contains a financial ETL pipeline and a Django-based REST API for the Nifty 100 stock universe.

- `data/` contains source Excel input files plus generated raw/clean CSV datasets.
- `etl/` contains the Python ETL pipeline for extraction, transformation, loading, and ML score generation.
- `nifty100/` is the Django project.
- `intelligence/` is the Django app exposing API endpoints and Celery tasks.
- `docker-compose.yml` starts PostgreSQL, Redis, Django, Celery worker, and Celery beat.

## Setup

### 1. Create a virtual environment

```bash
python -m venv .venv
.venv\Scripts\activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Start infrastructure services

To start PostgreSQL and Redis only:

```bash
docker-compose up -d postgres redis
```

To start the full stack with Django and Celery via Docker:

```bash
docker-compose up --build
```

---

## ETL Pipeline

Run the ETL steps in order after placing the source Excel files under `data/source/`.

```bash
python etl/01_extract_from_excel.py
python etl/02_clean_and_transform.py
python etl/03_load_to_warehouse.py
python etl/04_ml_scores.py
```

These scripts will:
- extract Excel data into raw CSV files,
- clean and normalize financial tables,
- load the warehouse into PostgreSQL,
- compute machine learning health scores.

---

## Run Django Locally

If you prefer local Django development instead of Docker:

```bash
set DJANGO_SETTINGS_MODULE=nifty100.settings
set DB_HOST=localhost
set DB_PORT=5433
set DB_NAME=nifty100
set DB_USER=postgres
set DB_PASSWORD=postgres
set DJANGO_DEBUG=True
```

Then run:

```bash
python manage.py migrate --run-syncdb
python manage.py runserver
```

## Environment Variables

The Docker Compose service already configures these defaults for local development:

- `DJANGO_SETTINGS_MODULE=nifty100.settings`
- `DJANGO_DEBUG=False`
- `DJANGO_SECRET_KEY` (override for production)
- `DB_HOST=postgres`
- `DB_PORT=5432`
- `DB_NAME=nifty100`
- `DB_USER=postgres`
- `DB_PASSWORD=postgres`
- `CELERY_BROKER_URL=redis://redis:6379/0`
- `CELERY_RESULT_BACKEND=redis://redis:6379/0`
- `NIFTY_DB_URL=postgresql://postgres:postgres@postgres:5432/nifty100`

For direct local development, use host-based values such as `localhost` and port `5433`.
